# The Problem
The PDF download functionality fails with a generic error alert: "عذراً، حدث خطأ أثناء تجهيز الملف".
**Diagnosis:** This is a classic **Replit Asset Loading Issue**. The `@react-pdf/renderer` library is failing to load the font files (`Cairo-Regular.ttf`) from the `/public` directory because Replit's dev server often mishandles serving static assets for PDF generation during runtime.

# The Definitive Solution (Base64 Embedding)
Do NOT try to fix the file path string. Instead, we must **embed the font directly into the code as a Base64 string**. This eliminates the network request entirely and guarantees the font loads.

# Task: Refactor `SanadReportPDF.jsx`
